# 7 - Run 'git --version' in your terminal. What is the output?
    It should display the version of Git you have installed.

# 8 - Why would you run the command 'git status'?
    git status is run to check the current state of the repository. 
    It shows which changes have been staged, which haven’t, and which files aren’t being tracked by Git.

# 9 - What does the command 'git add .' do?
    git add . stages all changes in the directory for the next commit.
    It includes both new files and modified existing files.

# 10 - What is the command to push your committed changes to your repo on GitHub?
    git push origin main
